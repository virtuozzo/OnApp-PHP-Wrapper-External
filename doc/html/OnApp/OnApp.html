API Wrapper for OnApp

  • todo:

    Pack using the lib (http://pecl.php.net/)

    The wrapper is used to describe the following basic methods: load, save, delete and getList.

    To create a new class inheriting this one, re-define the following variables:

    1.     // root tag used in the API request
    2.     var $_tagRoot  = '<root>';
    3.  
    4.     // alias processing the object data
    5.     var $_resource = '<alias>';
    6.  
    7.     // the fields array used in the response and request to the API server
    8.     var $fields   = array(
    9.      ...
    10.     )

    To create a read-only class, close the save and delete methods. To re-define the traditional API aliases to the non-traditional, re-define the getResource, getResourceADD, getResourceEDIT, getResourceLOAD, getResourceDELETE and getResourceLIST methods in the class that will be inheriting the ONAPP class.

    This API provides an interface to onapp.net allowing common virtual machine and account management tasks

    Usage OnApp_VirtualMachine class example ( Could be applied almost to any of the Wrapper classes ):

    Important ( OnApp CP Permissions Set Up):

    1.      Go to OnApp CP.
    2.      Users and Groups -> Roles
    3.      Push pencil edit icon to edit role of the user which you are going to use.
    4.      Check checkbox { View OnApp version (settings.version) }
    5.      Check other permissions in order to perform particular actions.

    Code example:

    Require Wrapper AutoLoad Class:

    1.     require_once '{Path to the Wrapper}/OnAppInit.php';

    Get OnApp Instance:

    1.      $onapp = new OnApp_Factory('{IP Address / Hostname}', '{Username}', '{Password}');

    Get OnApp_VirtualMachine Instance:

    1.      $vm_obj = $onapp->factory('VirtualMachine', {debug mode boolean, not required} );

    Get the list of VMs:

    1.      $vms    = $vm_obj->getList();

    Get particular VM:

    1.      $vm     = $vm_obj->load({VM ID});

    Create a VM:

    1.      $vm_obj->_label               = '{VM Label}';
    2.      $vm_obj->_memory              = {VM RAM };
    3.      $vm_obj->_cpu_shares          = {VM CPU priority};
    4.      $vm_obj->_hostname            = '{Hostname}';
    5.      $vm_obj->_cpus                = {number of VM CPUs};
    6.      $vm_obj->_primary_disk_size   = {VM Disk Space};
    7.      $vm_obj->_swap_disk_size      = {VM Swap Size};
    8.      $vm_obj->_template_id         = {VM Template ID};
    9.      $vm_obj->_allowed_hot_migrate = {VM Hot Migrate Boolean Value};
    10.  
    11.      $vm_obj->save();

    Edit VM:

    1.      $vm_obj->_id = {VM ID};
    2.      $vm_obj->_{Field You Want To Edit} = {New Value};
    3.  
    4.      $vm_obj->save();

    Getting debug log ( depends on debug mode ):

    1.      print_r( $vm_obj );

    For full fields reference and curl request details visit: ( http://help.onapp.com/manual.php?m=2 )
